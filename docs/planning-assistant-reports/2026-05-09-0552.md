## 二、程式運行模糊處
### 1. 探索進度有方向，但沒有轉層判定規則
- 位置：`docs/game-design.md` 173-199、1044；`docs/data/scenes/scenes.json`
- 模糊點：目前只寫了「探索進度增加」「可往下一層」「每次離開後歸零」，但沒有定義每層要累積多少進度才能進入下一層，也沒有定義 `currentArea`、`explorationProgress`、事件池、敵人池之間怎麼對應。
- 程式需要明確知道：玩家現在是在外圍、中層還是深處；`exploreDeeper` 每次增加多少；何時從外圍切到中層；失敗前進、迷路、無效前進是否消耗同樣成本；`forage` 與 `shoutAttractEnemy` 要從哪一層的資源池與敵人池抽取。
- 建議補規則：先把探索狀態拆成 `currentLayer` 與 `layerProgress` 兩個欄位。第一版可直接定義每層 `100` 進度，`exploreDeeper` 依結果加 `20 / 40 / 60`，滿 `100` 才切到下一層，切層後新層進度重設為 `0`。

### 2. 近戰判定缺少基底戰鬥力與不可用選項規則
- 位置：`docs/game-design.md` 205-229、268-270；`docs/data/enemies/enemies.json`
- 模糊點：目前只定義「有效近戰武器 +2」「近戰武器技能再 +1」，但沒有定義玩家的基底戰鬥力、空手是否能戰鬥、拿錯武器時是能打但很差，還是按鈕直接不出現。
- 程式需要明確知道：若玩家只有老舊的劍卻遇到怕槍的四足獸，是否仍可選擇近戰；若可選，基礎值是多少；若不可選，UI 是否要顯示 disabled 理由；體力過低時哪些戰鬥選項要被鎖住。
- 建議補規則：第一版把近戰拆成三態即可。`正確武器` 可用且套用 `base 1 + weapon 2 + skill 1`；`錯誤武器` 可用但只有 `base 1 + skill 1`；`空手` 直接禁用。再補一條最低體力門檻，例如體力 `0` 時不可主動近戰、體力 `1` 時不可使用士兵長槍。

### 3. 遠程削弱可零時間無限嘗試，會把高風險遭遇洗成穩定解
- 位置：`docs/game-design.md` 231-239
- 模糊點：遠程削弱目前不消耗時間，且可在進入近戰前反覆嘗試直到敵人難度降到 `0`。只要玩家攜帶足夠彈藥，實際上會把很多遭遇變成「多點幾次就過」。
- 程式需要明確知道：每次遠程削弱失敗後，敵人是否反擊、是否提高後續近戰難度、是否限制最多嘗試次數，或是否直接進入近戰。
- 建議補規則：第一版建議每場遭遇最多遠程削弱 `1` 次。成功則難度 `-1`；失敗則立即進入近戰，並額外體力 `-1`。如果想保留多次嘗試，也至少要讓每次嘗試消耗時間並提高受傷風險。

### 4. 指令已定義，但探索資源池與遭遇池尚未有明確資料責任
- 位置：`docs/data/commands/commands.json` 92-116；`docs/game-design.md` 199、245-283、475；`docs/data/scenes/scenes.json`
- 模糊點：`forage` 目前只有 `items: "regionLoot"`，`shoutAttractEnemy` 只有強制遭遇，但 repo 內還沒有對應的森林資源池、區域遭遇池或事件池資料表。
- 程式需要明確知道：外圍、中層、深處各自可能採到什麼、主動吸怪會抽哪些敵人、一般探索事件與敵人遭遇的優先順序是什麼。
- 建議補規則：不要先把它們塞回 `scenes.json`。第一版直接新增最小資料層，例如 `forest-regions.json`，每層只放 `resourcePool`、`enemyPool`、`eventPool`、`travelCostProfile` 四組欄位，這樣 `commands.json` 與 `enemies.json` 才有實際落點。

## 優先處理
1. 先補「探索層級 / 進度 / 轉層」規則，否則探索主循環沒有可執行節點。
2. 再補「近戰 / 遠程 / 逃跑」三種遭遇處理的判定邏輯，避免 UI 能出選項但後端不知道怎麼算。
3. 最後補區域資源池與遭遇池資料責任，讓 `forage`、`exploreDeeper`、`shoutAttractEnemy` 都能對應到資料。

## 三、五點規劃建議
1. 把探索狀態最小化成 `currentLayer`、`layerProgress`、`encounterState`、`temporaryLoot` 四塊，先不要再加熟練度或地圖記憶。
2. 把遭遇流程固定成 `發現敵人 -> 遠程削弱（可選） -> 近戰或工具處理 -> 逃跑或結算`，避免每隻怪各走一套隱藏流程。
3. 把敵人資料從「靜態特徵」與「遭遇行為」拆開思考；目前 `enemies.json` 已有靜態特徵，下一步缺的是遭遇用資料池，而不是再擴敵人名單。
4. 先定義外圍森林的一層完整閉環，再把同一套規則複製到中層與深處；不要三層一起補，否則你會先碰到平衡噪音而不是規則問題。
5. 把遠程削弱當成「開戰前賭一次優勢」而不是主要擊殺手段，這樣探索的準備壓力、補給壓力與怪物記憶才會成立。
