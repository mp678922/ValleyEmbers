# 企劃助手報告 - 2026-05-09 企劃助手（探索、戰鬥、煙霧彈）

結論：`煙霧彈` 很適合加入目前的遭遇系統，而且功能定位清楚，就是「降低當前危險度的主動保險工具」。這和你前面定的「使用道具不屬於危險動作」完全相容。

這次聚焦範圍：
1. `煙霧彈` 在探索遭遇規則中的作用
2. `煙霧彈` 與危險度的互動
3. `煙霧彈` 的狀態歸屬

我先採用的假設：
1. `煙霧彈` 是可消耗道具。
2. `使用道具` 仍然不觸發危險度判定，因此使用煙霧彈本身不會被怪物打斷。
3. `煙霧彈` 只能影響目前探索中的 `dangerRate`，不影響探索進度、敵人池、資源池與層級。
4. 危險度最低不會降到負數，因此若目前危險度低於 `30%`，使用後直接歸 `0%`。

建議規則草案：
- 觸發：
  玩家在探索區使用 `煙霧彈` 時生效。
- 可用條件：
  只要玩家目前位於探索區，且背包內有 `煙霧彈`，即可使用。
- 消耗：
  消耗 `煙霧彈 x1`。時間成本暫時沿用一般 `useItem` 對應的道具使用時間。
- 結果：
  目前危險度直接降低 `30%`。例如 `58% -> 28%`，`25% -> 0%`。
- 失敗或例外：
  `煙霧彈` 不能取消已經進入中的怪物遭遇。也就是說，若危險動作已經判定觸發遇敵並進入遭遇流程，就不能靠煙霧彈把這次遭遇撤銷；它只能在遭遇發生前降低下一個危險動作的遇敵率。
- 狀態歸屬：
  `煙霧彈` 是道具資料，放在道具表；目前降低的 `dangerRate` 屬於本次探索中的暫時狀態，放在探索存檔狀態內。

還需要你決定：
1. `煙霧彈` 是否只限探索區使用，還是戰鬥中也能作為逃脫道具。
2. `煙霧彈` 的重量、價格、是否可從蜥蜴羅索購得。
3. `煙霧彈` 是否要標記為一般 `tool`，還是新增更明確的脫逃/控場型標籤。

五條協助發想：
1. 如果你想讓逃跑與保險有分工，`逃跑` 處理「已經撞怪」，`煙霧彈` 處理「還沒撞怪前降風險」，兩者定位就很乾淨。
2. `煙霧彈` 很適合做成中期補給，而不是一開始就大量取得，這樣玩家才會思考何時值得保命。
3. 若之後想讓探索技能更有層次，可以讓高探索技能降低煙霧彈的浪費率，例如效果從固定 `-30%` 提升到 `-35%`，但第一版不必急著做。
4. 如果未來加入夜晚或濃霧區域，煙霧彈可以延伸成讓某些敵人更難鎖定玩家，而不只是一條數值下降。
5. 若之後覺得危險度系統太硬，可以把煙霧彈改成「降低目前危險度，並讓下一次危險動作不吃每分鐘累加」，但第一版先不要加複合效果。
