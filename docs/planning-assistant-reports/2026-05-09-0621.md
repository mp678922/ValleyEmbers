# 企劃助手報告 - 2026-05-09 企劃助手（探索、戰鬥、遭遇規則）

結論：這套方向可以成立，而且和你前面定下來的探索進度、戰鬥結算、危險動作邏輯是相容的。核心關鍵是把 `危險度` 定義成「下一個危險動作前的遭遇率」，並明確規定它的重算、累加、清零時機。

這次聚焦範圍：
1. 探索中的敵人遭遇規則
2. 危險度的生成、累加、清空
3. 危險動作與非危險動作的差異

我先採用的假設：
1. `危險度` 以百分比表示，最小 `0%`、最大 `100%`。
2. 每次「更新探索進度」後，都立刻重新隨機決定一次危險度，即使本次進度增加為 `0%` 也一樣。
3. `使用道具` 暫定為唯一不觸發遭遇判定的安全動作。
4. `等待`、`向深處探索`、`尋找物資`、`大叫吸引敵人`、未來其他探索動作，暫時都視為危險動作。
5. 戰鬥結束後危險度直接歸零，不保留先前累積值。

建議規則草案：
- 觸發：
  每次探索進度更新後，系統重新生成一次目前進度區段的 `危險度`。之後玩家每做一次危險動作，都先用目前危險度判定是否遇敵。
- 可用條件：
  只要玩家處於探索區且目前沒有正在處理中的遭遇事件，就持續啟用危險度系統。
- 消耗：
  危險動作照原本時間規則消耗時間；若玩家在同一探索進度下沒有推進到新進度，而是持續做其他指令，危險度每經過 `1` 分鐘增加 `1%`。
- 結果：
  若危險動作判定命中危險度，立刻進入敵人遭遇。若未命中，該動作正常結算。`大叫吸引敵人` 若未觸發戰鬥，則在原本時間推進與每分鐘累加之外，再額外提高一次危險度。
- 失敗或例外：
  `使用道具` 不做危險度判定，也不因使用道具本身提高危險度。戰鬥結束後，不論勝負，只要遭遇流程結束，危險度直接歸零。之後若玩家繼續探索，要等下一次更新探索進度或在同進度下繼續耗時間後，再重新累積危險度。
- 狀態歸屬：
  存檔中的探索狀態至少需要保存 `currentLayer`、`layerProgress`、`dangerRate`、`lastDangerProgress`、`encounterState`。其中 `dangerRate` 是目前下一次危險動作要用的遭遇率；`lastDangerProgress` 用來判斷玩家是不是仍停留在同一進度區段。

還需要你決定：
1. 每次「更新探索進度」時，危險度的隨機範圍要不要依森林層級不同而不同，例如外圍 `5%~20%`、中層 `15%~35%`、深處 `25%~50%`。
2. `大叫吸引敵人` 如果沒觸發戰鬥，要額外提高固定值，還是額外再擲一次小幅隨機值。
3. 同一進度下每分鐘 `+1%` 是否需要上限，例如最高只累到 `80%` 或 `100%`。

五條協助發想：
1. 可以把危險度理解成「當前區段的躁動程度」，不是地圖固定數值，而是玩家這趟探索把這一帶弄得多危險。
2. 若之後想做更細，危險度來源可以拆成兩段：`基礎危險度` 由層級決定，`追加危險度` 由玩家動作與停留時間累加。
3. `大叫吸引敵人` 很適合做成高風險高效率工具：平常可能失手，但會快速把危險度推高，方便農素材玩家主動控場。
4. 若未來覺得等待太容易被濫用，可以把 `等待` 的危險度累加係數調高，例如每分鐘 `+2%`，而不是改動全部危險動作。
5. 若之後想讓探索技能更有存在感，可以讓它不只影響進度效率，也小幅壓低每次重算出的基礎危險度。
